# Create a gradual onboarding plan that covers the first weeks

Create an onboarding plan that spans at least the first **four weeks**, and make sure you have small wins every week.

If you are a manager, you can design the plan together with the other engineers that will work with the new hire. You may assign tasks of increasing complexity, or that require increasing context about processes and tools within the company.

It is helpful to keep a list of small tasks ready for this, like the *#first-friendly* ones we discussed before.

Here is an example of 30-days [onboarding plan developed by HubSpot](https://gist.github.com/jdwyah/a84f36a37f617d6d4cc3).
