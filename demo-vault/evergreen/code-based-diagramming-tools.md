# Code-based diagramming tools

A couple of weeks ago we talked of [diagramming tools](https://refactoring.fm/i/73452481/diagramming-tools). These may be handy, but one common limitation is that diagrams are cumbersome to update and are not version-controlled.

Enter code-based diagramming tools, that transform code into actual diagrams.

Last week, [Pablo](https://hawkyre.com/) surveyed the community about what are the best tools for this, and we got some great responses:

- [DB Diagram](https://dbdiagram.io/home) — for creating ER diagrams.

- [Graphviz](https://en.wikipedia.org/wiki/Graphviz) — the old-school open source solution! Many tools have been built on top of it — you can find them from the linked page. 

- [Diagram.codes](https://www.diagram.codes/) — one of the most flexible ones out there. You can use it to create graphs, mindmaps, ER, and much more.

- [PlantUML](http://www.plantuml.com/plantuml/uml/SyfFKj2rKt3CoKnELR1Io4ZDoSa70000) — for sketching sequence flow diagrams

- [Cloudcraft](https://www.cloudcraft.co/) — for creating gorgeous cloud architecture diagrams. You can also directly import your AWS environment and keep diagrams updated automatically. Neat!
