# The vicious vs virtuous cycle of docs

Writing good docs is tricky.

In my experience, most companies have either 1) a good workspace for docs, reasonably updated, where everyone contributes to, or 2) basically no docs at all, or just very outdated ones.

There is hardly a middle ground. 

I think that’s because documentation naturally leads to either a virtuous or vicious cycle.

When future value is higher than writing effort, you get a 🟢 **virtuous cycle**:

1. People write more docs because they feel they are valuable

1. Knowledge grows larger, more insights are discovered, decisions are taken faster and better

1. People read more and more docs instead of having meetings

When future value is lower than writing effort, you get a 🔴 **vicious cycle**:

1. People stop writing docs because no one reads them

1. Docs become partial and outdated

1. People don't trust and read docs anymore

**Writing docs is like writing tests** — people do it if the value of future use is perceived as superior to the writing effort. So to write more docs you can either (or both):

1. **Increase future use** — by embedding them in processes, turn meetings into shared notes, etc.

1. **Reduce writing effort** — with processes, templates, clear structure.
