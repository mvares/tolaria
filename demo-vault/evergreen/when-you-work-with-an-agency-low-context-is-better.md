# When you work with an agency → Low-context is better

Things are easier if your partner doesn't need too much context and onboarding to get started. There are many types of *context*:

- **Product context **— how much business / domain knowledge is required to build the product? How much learning is required to do a good job?

- **Design context **— do you have a design language / system to build on, or you need to create one for this project?

- **Tech context **— does the project plug into existing tech? Does your partner need to find their way into your code?

Even if some kind of context is inevitable, reflect on how much onboarding the agency will need with respect to the expected size of the work. 

If the project is too much ingrained in your workflows and has too many dependencies, working with a partner might not work out as you would like.
